# 🧠 AetherFlow: Intelligent Query Routing with FastAPI

AetherFlow is an advanced FastAPI-based service that analyzes, classifies, and intelligently routes user queries to the most suitable AI backend. It leverages machine learning (KMeans clustering, TF-IDF vectorization), statistical anomaly detection, risk scoring, and semantic intent analysis to decide whether a query should be handled by a fast cache, a heavy LLM, or be blocked for safety.

---

## 🚀 Features

- **Text Classification:** Uses TF-IDF and KMeans to categorize queries (General Knowledge, Programming, Mathematics).
- **Anomaly Detection:** Calculates Mahalanobis distance to flag outlier queries.
- **Risk Scoring:** Bayesian risk score based on sensitive keywords.
- **Semantic Intent Routing:** Detects if a query requires deep reasoning or can be answered quickly.
- **Rich Console Output:** Beautiful, color-coded query analysis in the terminal using `rich`.
- **Ollama Integration:** Routes queries to different LLMs (e.g., TinyLlama, Llama3) via Ollama API.
- **REST API:** Exposes a `/route` endpoint for programmatic access.
- **Extensible:** Easily add more clusters, risk words, or routing logic.

## ⚡ Quickstart

### 1. Clone the Repository

```bash
git clone https://github.com/SIDR1921/aetherrouter.git
cd aetherrouter
```

### 2. Set Up a Virtual Environment

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```
> If `requirements.txt` is missing, install manually:
```bash
pip install fastapi uvicorn numpy scikit-learn pydantic rich requests sentence-transformers
```

### 4. Start Ollama

Make sure [Ollama](https://ollama.com/) is running locally and the models (`llama3.2`, `tinyllama`) are available.

### 5. Run the FastAPI Server

```bash
uvicorn main:app --reload
```

### 6. Access the API & Docs

- Interactive API docs: [http://localhost:8000/docs](http://localhost:8000/docs)
- Query endpoint: `POST /route` with JSON body:
    ```json
    {
      "prompt": "who wrote hamlet"
    }
    ```

## 🧩 How It Works

1. **Receives a Query:** User sends a prompt to `/route`.
2. **Analyzes Query:**
   - Calculates entropy (complexity).
   - Computes anomaly score (Mahalanobis distance).
   - Assigns a risk score (based on sensitive words).
   - Classifies topic using KMeans clustering.
   - Checks for deep-thinking intent.
3. **Routing Decision:**
   - **Blocked:** If anomalous or high risk.
   - **Fast Cache:** If simple and short.
   - **Heavy LLM:** If complex or requires reasoning.
4. **LLM Response:** For routed queries, fetches a response from the appropriate Ollama model.
5. **Rich Console Output:** Prints a color-coded analysis table in your terminal.
6. **Returns JSON:** With status, metrics, routing decision, and AI response.

## 🧑‍💻 Development & Customization

- **Add More Training Data:** Edit `training_texts` in `main.py` for better clustering.
- **Tune Risk Words:** Update the `RISKWORDS` set.
- **Change Routing Logic:** Modify the `/route` endpoint logic for your use case.
- **Integrate More LLMs:** Adjust `query_ollama` to support more models.
